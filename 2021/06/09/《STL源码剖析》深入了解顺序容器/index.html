顺序容器概述

顺序容器类型 \
vector 可变大小数组，支持快速随机访问。在尾部之外的位置插入或删除元素可能很慢。
deque 双端队列。支持快速随机访问，在头尾位置插入、删除很快。
list 双向链表。只支持双向顺序访问。在list中任何位置插入、删除速度都很快。
forward_list 单向链表。只支持单向顺序访问。在list中任何位置插入、删除速度都很快。
string 与vector相似的容器。但专门用于保存字符。随机访问快。在尾部插入、删除速度快。

标准库容器都提供了高效、灵活的内存管理。我们可以添加和删除元素，扩张可收缩容器的大小。容器保存元素的策略对容器操作的效率有着很大的影响，甚至影响特定容器是否支持特定操作。
例如vector和string将元素保存在连续的内存空间中。由于元素是连续存储的，所以通过下标来计算地址是非常快速的，但是在这两种容器的中间位置插入或删除元素就会非常耗时：在一次插入或删除操作后，需要移动插入/删除位置之后的所有元素，来保持连续存储。而且，添加元素还可能需要分配额外的空间，再将所有元素整体移动到新的存储空间中。
list和forward_list容器的设计目的是令容器任何位置的添加和删除操作都很快速。作为代价，不支持元素的随机访问：为了访问一个元素，只能遍历整个容器。而且相比于vector、deque额外开销更大。

下面我们对具有代表性的三种顺序容器展开具体讨论。

vector

vector实现的关键在于对大小的控制以及重新配置时数据移动的效率。一旦vector就有空间满载，如果客户端增加元素，则扩充空间，为了防止扩充空间导致的数据频繁整体移动，所以加入了未雨绸缪的策略，即每次扩充足够大的空间（一般为原先的两倍）。在空间利用率和时间效率两者之间折中。

vector的迭代器

vector维护的是一个连续线性空间，所以无论其类型是什么，普通指针都可以作为vector的迭代器而满足所有必要条件，因为vector迭代器所需要的操作行为，如operator*,operator->，operator++，operator–，operator+，operator-，operator+=，operator-=，普通指针本来就具有。vector支持随机存取，而普通指针也有这样的能力。

为了降低空间配置的时间成本，vector实际配置的大小可能比客户端需求量更大一些，以便将来可能的扩充，这便是容量的观念。换句话说，一个vector的容量永远大于或等于其大小。一旦容量等于大小，便是满载，下次再有新增元素，整个vector就得寻找其他空间。

注意，所谓的动态增加大小，并不是在原空间周接续新空间（因为无法保证原空间之后尚有可配置的空间），而是以原大小的两倍（如果两倍容量仍不足，则扩张到足够大容量）另外配置一块足够大空间，然后将原内容拷贝过来，然后才开始在原内容之后构造新元素，并释放原空间。因此，对于vector的任何操作，一旦引起空间重新配置，指向原vector的所有迭代器都失效了。

list

相较于vector的连续线性空间，list就显得复杂许多，它的好处是每次插入或删除一个元素，就配置或释放一个元素的空间。因此，list对于空间的运用有绝对的精准，一点也不浪费。对于任何位置的元素的插入或元素删除，list永远是常数时间。

list的迭代器

list不再像vector一样以普通指针作为迭代器，因为其节点不保证在储存空间中连续存在。list迭代器必须有能力指向list的节点，并有能力进行正常的递增、递减、取值、成员存取等操作。递增时指向下一节点，递减时指向上一节点，取值是去的是节点的数据值，成员取用时取的是节点的成员。

deque

deque是双向开口的连续线性空间，deque允许在常数时间内对头端或尾端进行元素的插入或移除操作。deque没有容量的观念，因为它是动态以分段连续空间组合而成，随时可以增加一段新的空间并链接起来。换句话说，像vector那样“因旧空间不足而重新分配一块更大空间，然后复制元素，再释放旧空间”这样的事情在deque不会发生。
虽然deque也提供了随机访问的迭代器，但它的迭代器并不是普通指针。其复杂度相比于vector大很多，因此，除非必要，尽量选择vector而非deque。

deque的中控器

deque是由一段一段的定量连续空间构成。一旦有必要在deque的前端或尾端增加新空间，便配置一段连续空间，串接在整个deque的头端或尾端。deque的最大任务，便是在这些分段的定量连续空间上，维护其整体连续的假象，并提供随机存取的接口。避开了“重新配置、复制、释放”的轮回，代价则是复杂的迭代器架构。
deque采用一块所谓的map作为主控，map是一小块连续的空间，其中每个元素都是指针，指向另一段（较大的）连续线性空间，称为缓冲区，缓冲区才是deque的储存空间主体。
map是一个指针，所指之物又是一个指针，指向类型为T的一块空间。

deque的迭代器

deque是分段连续空间。维持其“整体连续”假象的任务，落在了迭代器operator++和operator–两个运算子身上。
deque的迭代器应该具备什么结构？首先，它必须能够指出分段连续空间在哪里，其次它必须能够判断自己是否已经处在其所在缓冲区的边缘。，如果是，一旦前进或后退就必须跳跃至下一个或上一个缓冲区。
